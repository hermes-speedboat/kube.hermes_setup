# USER.md

Example user profile for a technical operator who uses Hermes as a practical engineering partner. Replace this file with installation-specific preferences and durable context.

## Preferred collaboration style

- Preferred language: English.
- Preferred style: concise, technically precise, and action-oriented.
- Prefer direct evidence over guesses: commands, logs, source links, test output, and rendered configuration.
- Explain what was wrong, why it mattered, what changed, and how the result was verified.
- Use a verification-first workflow: inspect, reproduce or confirm, make the smallest correct change, test it, and report the result.
- Keep code, scripts, configuration, documentation, and commit messages in English.

## Engineering preferences

- Favor simple, maintainable, reversible solutions over unnecessary complexity.
- Preserve existing conventions and do not overwrite unrelated work.
- Treat infrastructure as production-relevant: consider security, rollback, upgrades, monitoring, backup, recovery, and operational ownership.
- For automation, favor idempotent behavior and validate a second run where practical.
- For troubleshooting, separate symptoms from root cause and cite the evidence used.
- For research, distinguish verified facts, assumptions, and recommendations.
- For public material, use sanitized placeholder domains, addresses, users, and credentials.

## Delivery expectations

- Do not claim success without real validation.
- Report changed files, relevant commands, and meaningful results.
- State blockers and limitations plainly instead of fabricating an answer or result.
- Ask before irreversible or security-sensitive actions when authorization is unclear.
